web开发中的复制粘贴

发表于 2024-07-05 更新于 2024-08-14 阅读次数：
在web开发中，经常会遇到复制粘贴的需求，这里做一个总结。

写入剪贴板

写入剪贴板有三种方式：

  • 使用document.execCommand
  • 使用Clipboard API
  • 键盘组合键Ctrl+C、Ctrl+X

我们一个个来看。

document.execCommand

document.execCommand是一个很古老的 API，目前已经被废弃。虽然大多数浏览器为了兼容老网站还没有移除对它的支持，但绝对不再鼓励使用这个函数，这里只简单介绍一下。

写入剪贴板可使用copy和cut命令。可以访问这个example来查看浏览器对命令的支持。

用法是先选中要复制的内容，然后调用document.execCommand("copy")或document.execCommand("cut")。

Clipboard API

使用 Clipboard API 是最推荐的做法，主流浏览器全部支持这个 API。

写入剪贴板（Edge 使用“剪贴板”这个翻译）有两个接口：writeText和write。

其中，writeText只能写入 plaintext 文本，而write可以写入任意 MIME 类型的数据。

除了标准的 MIME 类型数据外，write还支持写入web 自定义格式的数据：
在 MIME 类型前面添加web 前缀，来避免与标准 MIME 类型冲突。

但注意，写入web自定义格式需要请求clipboard-write 权限。

组合键

组合键的写入权限是最强的，因为这是用户非常明确的操作，用户非常清楚Ctrl+C按下去会发生什么。

当用户按下Ctrl+C时，浏览器会触发copy事件，传入一个ClipboardEvent。copy 事件会像标准事件一样，逐步走过捕获、目标、冒泡阶段。

event.clipboardData是一个DataTransfer对象，可以通过DataTransfer.setData方法写入数据。

像之前说的，组合键的功能是最强大的。在 copy 事件处理中，我们除了可以写入MIME类型的数据外，还可以写入任意自定义格式的数据。

读取剪贴板

我们分别来看看和写入对应的三种方式：

  • 使用document.execCommand
  • 使用Clipboard API
  • 键盘组合键Ctrl+V

document.execCommand

document.execCommand并不能直接读取剪贴板中的内容，使用paste可以将剪贴板中的text/plain类型数据直接写入到contenteditable的 element 中。

但出于安全考虑，目前所有的主流浏览器都会禁止paste命令。

可使用queryCommandSupported来查看浏览器是否支持paste命令。

Clipboard API

要使用 Clipboard API 读取剪贴板，一定要先申请clipboard-read 权限。

注意，clipboard.read只能读取 MIME 类型以及web + MIME 类型的数据，不能读取自由格式的数据。

组合键

当用户按下Ctrl+V时，浏览器会触发paste事件，传入一个ClipboardEvent。

注意，paste事件无法读取到web + MIME 类型的数据。
